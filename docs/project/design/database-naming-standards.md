# Database Naming Standards

**Project:** Aarvii CCTV AMC Management System
**Phase:** D0-4 — Entity Model, ER Diagram & Database Architecture (design only)
**Applies to:** all CCTV business schemas (`cctv_*`). Platform schemas are frozen and out of scope.

---

## 1. General conventions

| Item | Standard | Example |
|------|----------|---------|
| Case | `snake_case`, lowercase, ASCII only | `amc_contract_terms` |
| Schemas | `cctv_<module>` | `cctv_service` |
| Tables | **Plural** snake_case of the entity name | `Lead` → `leads`; `SiteAssetSummary` → `site_asset_summaries` |
| Columns | Singular snake_case; no table-name prefixes | `status`, `scheduled_date` |
| Reserved words | Never used as identifiers | use `description`, not `desc` |
| Abbreviations | Only well-known domain ones (`amc`, `gps`, `nvr`, `dvr`) | `amc_plans` |

## 2. Primary keys

| Rule | Standard |
|------|----------|
| Column | `id UUID` (v7/sequential-friendly) on every table |
| Constraint name | `pk_<table>` → `pk_leads` |
| Business identifier | Separate human-readable number column with unique index (never the PK): `lead_number`, `customer_number`, `site_number`, `contract_number`, `schedule_number`, `ticket_number`, `invoice_number` |
| Number format | `<PREFIX>-<YYYY>-<sequence>` (e.g. `LD-2026-0001`, `INV-2026-0042`); generated by the owning module |

## 3. Foreign keys

| Rule | Standard |
|------|----------|
| Column name | `<referenced_entity_singular>_id` → `lead_id`, `service_visit_id`, `amc_contract_term_id` |
| Physical FK (same schema only) | `fk_<child_table>_<parent_table>` → `fk_ticket_comments_tickets` |
| Cross-schema / platform references | Same column naming, **no physical constraint** (logical reference; see [database-architecture.md §3](./database-architecture.md)) — e.g. `file_id`, `customer_id` (from `cctv_ticket`), `plan_version_id` |
| Delete behavior | Composition children: `ON DELETE CASCADE` only where the root is hard-deletable; since roots are soft-deleted (see §7), default is `ON DELETE RESTRICT` |
| Actor references | `created_by`, `updated_by`, `assigned_by`, `reviewed_by`, `changed_by` — UUID of the platform Auth user; logical reference |

## 4. Indexes

| Rule | Standard |
|------|----------|
| Naming | `ix_<table>_<column[_column]>` → `ix_tickets_site_id_status` |
| FK columns | Every FK/logical-reference column is indexed |
| Status/lookup columns | Indexed where filtered by lifecycle (e.g. `ix_service_schedules_status`, `ix_invoices_status`) |
| Date-range queries | Index scheduling/expiry dates: `ix_service_schedules_scheduled_date`, `ix_amc_contract_terms_end_date` (AMC expiry reminders) |
| Soft delete | Hot-path indexes are partial: `WHERE is_deleted = false` |

## 5. Unique constraints

| Rule | Standard |
|------|----------|
| Naming | `ux_<table>_<column[_column]>` → `ux_customers_customer_number` |
| Business numbers | Unique per table (`ux_leads_lead_number`, …) |
| Conditional (partial) uniques for business rules | `ux_amc_contracts_site_id_active` on `(site_id) WHERE status = 'Active'` — one active contract per site (BR-AMC-02) |
| | `ux_amc_contract_terms_contract_id_active` on `(contract_id) WHERE status = 'Active'` — one active term |
| | `ux_engineer_assignments_schedule_active` on `(service_schedule_id) WHERE is_active = true` |
| | `ux_ticket_assignments_ticket_active` on `(ticket_id) WHERE is_active = true` |
| 1:1 children | `ux_site_asset_summaries_site_id`, `ux_service_visits_service_schedule_id`, `ux_visit_locations_service_visit_id`, `ux_visit_signatures_service_visit_id` |
| Max-3 contacts (BR-STRUCT-03) | `contact_slot smallint NOT NULL CHECK (contact_slot BETWEEN 1 AND 3)` + `ux_site_contacts_site_id_contact_slot (site_id, contact_slot)` — DB guarantees ≤3 alongside the aggregate invariant |
| Versioning | `ux_amc_plan_versions_plan_id_version_no (amc_plan_id, version_no)`; `ux_amc_contract_terms_contract_id_term_no (amc_contract_id, term_no)` |
| Check constraints | `ck_<table>_<rule>` → `ck_tickets_status`, `ck_invoices_total_amount_non_negative`, `ck_visit_locations_lat_lng_range` |

## 6. Audit fields (mandatory on every table)

| Column | Type | Notes |
|--------|------|-------|
| `created_at` | `timestamptz NOT NULL` | UTC, set on insert |
| `created_by` | `uuid NOT NULL` | Platform user id (or system id for auto-generated rows, e.g. website lead, schedule generation) |
| `updated_at` | `timestamptz NULL` | UTC, set on every update |
| `updated_by` | `uuid NULL` | Platform user id |
| `row_version` | concurrency token | On aggregate-root tables |

Append-only tables (`lead_activities`, `*_status_histories`, `visit_approvals`) carry only `created_at` / `created_by` semantics (their rows are never updated).

> The full compliance audit trail lives in the platform Audit module (MongoDB); these columns are row stamps, not the audit log.

## 7. Soft delete standards

| Rule | Standard |
|------|----------|
| Columns | `is_deleted boolean NOT NULL DEFAULT false`, `deleted_at timestamptz NULL`, `deleted_by uuid NULL` |
| Applies to | Aggregate roots that admins can remove: leads, customers, sites, amc_plans, engineers, tickets (and their file-attachment rows) |
| Never deleted (not even soft) | `amc_contracts`, `amc_contract_terms` (permanent records, freeze §8), `amc_plan_versions` (immutable history), all `*_status_histories`, `visit_approvals`, `lead_activities`, completed `service_visits` and their evidence (legal/service record) |
| Hard delete | Forbidden for business data; only platform-level data-retention operations (out of V1 scope) |
| Query behavior | EF global query filters exclude `is_deleted = true`; partial indexes mirror the filter |
| Status vs deletion | Deactivation (`status = 'Inactive'`) is a business state, distinct from soft delete (admin removal of erroneous records) |

## 8. Enum/status storage

| Rule | Standard |
|------|----------|
| Storage | `varchar` with `ck_` CHECK constraints against the **frozen** status lists (BR-LEAD-01, BR-SCHED-01, BR-TKT-01/02, BR-INV-01) |
| Values | PascalCase tokens exactly as approved: `QuotationSent`, `InProgress`, `Reopened` … |
| Extension | New values require a change request (scope freeze §22) |

## 9. Money, geo, and time

| Data | Standard |
|------|----------|
| Money | `numeric(18,2)` (`decimal` in code — never float); columns suffixed `_amount` or `price` |
| GPS | `latitude numeric(9,6)`, `longitude numeric(9,6)`, `captured_at timestamptz` (BR-VISIT-02) + range checks |
| Dates vs instants | Business dates (term start/end, scheduled_date, invoice_date, due_date) = `date`; events/stamps = `timestamptz` (UTC) |

## 10. File references (mandated)

| Rule | Standard |
|------|----------|
| Column | `file_id uuid NOT NULL` → platform `files.FileRecord` (logical reference) |
| Naming | Always exactly `file_id`; one file per row (multi-file = multiple rows in the attachment table) |
| Forbidden | `*_path`, `*_url`, `*_blob` columns in any business table |
