## 1. Platform summary

Ashraak is an enterprise **multi-tenant SaaS starter platform**. V1 delivers a production-shaped foundation across three runtimes that share one identity and tenancy model:

- **Backend** — .NET 10 modular monolith (ASP.NET Core Minimal APIs, EF Core 9, OpenIddict), with per-module schemas on PostgreSQL, Redis caching, and MongoDB-backed audit.
- **Web** — React 19 + TypeScript SPA (Vite 6, TanStack Query, Zustand, react-router, react-hook-form + Zod) with a swappable **Theme Engine**.
- **Mobile** — Flutter foundation (Riverpod, go_router) mirroring the platform's modules.

The platform is designed to be **forked and extended**: new products start with Auth, Tenancy, Users, Audit, Notifications, Files, Webhooks, API Keys, caching, observability, and CI already wired.

## 3. Architectural achievements

- **Modular monolith** ([ADR-0001](../adr/ADR-0001-modular-monolith.md)) — modules communicate only through contracts; each owns its schema.
- **Observer modules** ([ADR-0003](../adr/ADR-0003-observer-modules.md)) — Audit observes events without coupling producers to it.
- **Outbox + hosted processors** ([ADR-0002](../adr/ADR-0002-outbox-pattern.md), [ADR-0007](../adr/ADR-0007-outbox-hosted-processors.md)) — reliable, transactional event publication.
- **Theme Adapter Architecture** ([theme decision record](../frontend/themes/theme-decision-record.md)) — UI themes are swappable adapters; business code never imports a theme.
- **Central package management & build governance** ([ADR-0010](../adr/ADR-0010-build-package-governance.md)) — single source of dependency versions.
- **Flutter mobile platform** ([ADR-0012](../adr/ADR-0012-flutter-mobile-platform.md)) — one mobile codebase across Android/iOS.

### Theme Engine
A permanent, governed capability. Business modules render `platform-ui` primitives against **11 typed contracts**; the active theme is resolved from `VITE_THEME` via a registry. Ships with **CoreUI** (production default) and **HexaDash** (validated, opt-in). Direct theme integration is forbidden. See the [theme governance docs](../frontend/themes/theme-decision-record.md).

### Webhooks
Event-driven outbound integration: subscriptions, a signed delivery engine, retry with backoff, dead-letter queue and replay, plus an admin operations center on web and mobile. ADRs [Webhook-0001..0004](../adr/ADR-Webhook-0001-webhook-platform-architecture.md).

### API Keys
Programmatic access: scoped key issuance, hashing, rotation, revocation, authentication middleware, usage tracking, and admin UI. [ADR-ApiKeys-0001](../adr/ADR-ApiKeys-0001-api-keys-platform.md).

### Files
Tenant-scoped storage with pluggable backends (local / S3 / Azure), upload/download, and URL issuance. [ADR-0011](../adr/ADR-0011-files-storage-module.md).

### Auth
Identity and access: registration, login, JWT/OAuth2 via OpenIddict, RBAC + ABAC permissions, MFA/TOTP, invitations, session management, and SSO foundations. [ADR-0009](../adr/ADR-0009-auth-tenant-completion.md).

### Mobile
Flutter foundation covering auth, tenant, users, profile, sessions, audit, API keys, webhooks, files, notifications, and settings — with secure token storage, offline cache, push notifications, crash reporting, analytics, and OpenAPI-generated SDK. Mobile ADRs [Mobile-0001..0012](../adr/ADR-Mobile-0001-state-management.md).

### Governance
Documentation governance (code + docs evolve together), the ADR catalog, module-map discipline, and the selection/onboarding rules for themes. Enforced in CI by documentation validation.

### Release Engineering
CI for backend/frontend (`ci.yml`), documentation validation (`docs-validation.yml`), and mobile build + store release pipelines (`mobile.yml`, `android-release.yml`, `ios-release.yml`). Mobile versioning, signing, and rollback strategies are codified ([Mobile-0010..0012](../adr/ADR-Mobile-0010-release-strategy.md)).

### Documentation Governance
Every feature ships with docs under `docs/`. A PR checklist and `validate-docs` script gate merges; an audit tracks scaffold-vs-implementation gaps. See [documentation-governance.md](../documentation-governance.md).
